## 阶段 0：架构升级（前置基础）

> **目标**：解决阻塞大部分技能完整实现的3个核心架构问题

### 0-A：PendingResponse 队列化 ✅ 已完成

**问题**：当前 `pendingResponse` 是单一对象，多个被动技能同时触发时只有第一个能执行。

**方案**：
1. `shared/src/index.ts`：将 `GameState.pendingResponse` 改为 `pendingResponseQueue: PendingResponse[]`
2. `GameStateClientView` 中 `pendingResponse` 改从队列取 `[0]`
3. `handleRespond` 处理完一个后 `shift()` 取下一个
4. `dealDamage` 中的反馈/遗计/刚烈不再互斥（去掉 `!state.pendingResponse` 检查）

**影响文件**：`shared/src/index.ts`, `game-state.ts`, `game-actions.ts`, `handlers.ts`, `GamePage.tsx`

**验证**：曹操（奸雄）+ 司马懿（反馈）同时受伤时，两个技能都能正确触发

---

### 0-B：濒死询问异步化 ✅ 已完成

**问题**：`enterDyingState` 是同步遍历自动找桃，不允许玩家选择。

**方案**：
1. 新增 `ResponseType.PEACH_SAVE_ASK`
2. `enterDyingState` 改为设置 `pendingResponse(PEACH_SAVE_ASK)`，自动轮转给逆时针下一个角色
3. `handleRespond` 中处理 PEACH_SAVE_ASK：出桃=救活，跳过=轮转下一人，全部跳过=死亡
4. 完杀（贾诩）仍在轮转时跳过非濒死角色
5. 救主（赵云）/急救（华佗）改为在 PEACH_SAVE_ASK 响应中可选发动

**验证**：濒死时所有角色依次被询问，玩家可手动选择是否出桃

---

### 0-C：判定介入机制（鬼才/缓释） ✅ 已完成

**问题**：判定直接生效，没有"判定牌生效前"的介入窗口。

**方案**：
1. 新增 `ResponseType.JUDGE_INTERVENE`
2. `runJudgePhase` 改为：翻判定牌 → 检查是否有鬼才/缓释持有者 → 若有则设置 `pendingResponse(JUDGE_INTERVENE)` → 等待响应后继续
3. 鬼才（司马懿）：打出手牌替换判定牌
4. 缓释（诸葛瑾）：己方判定前可打出牌替换
5. 天妒（郭嘉）：在判定最终生效后仍获得牌

**验证**：乐不思蜀判定时，司马懿/诸葛瑾可以用手牌替换判定牌

---

### 0-D：马术技能距离修正 ✅ 已完成

**问题**：`getAttackRange` 和 `getSeatDistance` 只看装备坐骑，不看马术/镇卫技能。

**方案**：
1. `game-state.ts` 的 `getAttackRange` 增加马术检查：马超/庞德计算距离-1
2. 新增 `getEffectiveDistance` 函数，整合：
   - 马术（马超/庞德）：自己到他人距离-1
   - 镇卫（文聘）：对方到己方其他角色距离+1
   - 装备坐骑效果
3. 所有距离相关函数统一调用 `getEffectiveDistance`

**验证**：马超不带-1马也能攻击距离2的角色

## 优先级建议

如果资源有限，建议按以下优先级：

1. **P0（必须）**：阶段 0-A（队列化）→ 这解锁绝大多数后续工作
2. **P0（必须）**：阶段 0-D（马术距离）→ 影响攻击范围正确性
3. **P1（高优先）**：阶段 1-2（修复逻辑偏差）→ 已有代码的 bug 修复
4. **P1（高优先）**：阶段 0-B（濒死异步）→ 核心游戏体验
5. **P2（中优先）**：阶段 0-C + 2-1（判定介入+鬼才缓释）
6. **P2（中优先）**：阶段 2-2（流离/天香）
7. **P3（低优先）**：阶段 3（神速/牌转换等复杂技能）
