# 三国杀 3v3 网页游戏 — 产品需求说明文档（PRD）

> 版本：v0.1 | 最后更新：2026-02-22

---

## 一、产品概述

### 背景
三国杀 3v3 模式（又称"统率三军"）是官方曾经提供的竞技模式，现已下架。本项目旨在提供一个可在网页端游玩的 3v3 对战平台，供玩家体验这一经典玩法。

### 核心定位
- **玩法**：2 名玩家对战，每人各自操控己方 3 个武将
- **平台**：网页端（PC 浏览器优先）
- **规模**：支持 100 人以内同时在线

---

## 二、用户需求

| 需求 | 说明 |
|------|------|
| 无需注册 | 玩家直接输入昵称，无需账号体系 |
| 快速匹配 | 通过房间码邀请好友进行 1v1 对战 |
| 完整 3v3 规则 | 遵循官方三国杀 3v3（统率三军）规则 |
| 实时对战 | 低延迟的实时游戏体验 |

## 三、游戏规则（官方 3v3）

### 3.1 基本设置

| 属性 | 说明 |
|------|------|
| 对战人数 | 2 名玩家（每人操控 3 个武将） |
| 阵营 | 暖色方（主公+忠臣×2） vs 冷色方（内奸+反贼×2） |
| 胜利条件 | 击杀对方主帅即获胜 |

### 3.2 座次顺序（固定）

```
冷色前锋A → 冷色主帅 → 冷色前锋B → 暖色前锋B → 暖色主帅 → 暖色前锋A
```
（按逆时针方向行动）

### 3.3 卡牌池

- 基于三国杀标准版卡牌
- **去除**：闪电
- **加入**：兵粮寸断 ×2
- 总计约 106 张

### 3.4 选将流程

1. 随机产生 16 张武将牌作为可选武将池（双方各贡献 8 张，由游戏系统实现）
2. 暖色方主帅决定先选方
3. 按 **1-2-2-2-2-2-2-2-1** 顺序轮流选将，每方最终各选 8 张
4. 选将结束后，各方在规定时间内秘密安排 3 张武将（主帅/前锋A/前锋B）
5. 双方同时亮出部署

> ⏱️ 选将与部署阶段是否加入倒计时：**待定**

### 3.5 行动顺序

- 冷色方主帅决定哪方先行动
- 有行动权的一方（玩家）选择本回合行动单元：
  - **主帅行动**：主帅执行完整一回合，移交行动权
  - **前锋行动**：两名存活前锋依次执行一回合，移交行动权
- 每个完整对战回合，双方各有 1 次主帅行动单元 + 1 次前锋行动单元

### 3.6 回合结构

每个武将的回合依次包含：
1. **回合开始阶段** — 触发技能
2. **判定阶段** — 处理判定区延时锦囊
3. **摸牌阶段** — 摸 2 张牌
4. **出牌阶段** — 使用手牌/技能（单回合限用 1 张杀）
5. **弃牌阶段** — 手牌数降至当前体力
6. **回合结束阶段** — 触发技能

### 3.7 特殊规则

| 规则 | 说明 |
|------|------|
| 击杀奖励 | 击杀任意角色（含友军）立即摸 3 张牌 |
| 主公技 | 本模式不使用 |
| 闪电 | 本模式不使用 |
| 南蛮入侵 / 万箭齐发 | 出牌顺序可由出牌方指定（顺/逆时针） |

### 3.8 胜负判定

1. 对方主帅阵亡 → 立即获胜
2. 超时（若启用时限）→ 存活角色多的一方获胜；相同则比较已损失体力，少者获胜

## 四、功能需求

### 4.1 大厅系统
- [ ] 输入昵称（2~12 个字符）
- [ ] 创建房间（生成 6 位房间码）
- [ ] 加入房间（输入房间码）
- [ ] 房间等待界面（等待对手加入）

### 4.2 选将系统
- [ ] 展示武将池（16 张）
- [ ] 按选将顺序高亮轮次提示
- [ ] 选将阶段完成后的部署界面
- [ ] 选将/部署倒计时（**待定**）

### 4.3 游戏系统
- [ ] 行动单元选择（主帅 / 前锋）
- [ ] 当前武将的手牌展示与操作
  - [ ] 使用基本牌（杀、闪、桃、酒）
  - [ ] 使用锦囊牌
  - [ ] 装备牌安装/替换
- [ ] 武将技能发动
- [ ] 目标选择（攻击范围高亮）
- [ ] 响应阶段（出闪、出桃、无懈可击等）
- [ ] 判定展示
- [ ] 行动日志

### 4.4 结算系统
- [ ] 游戏结束弹窗（胜/负方）
- [ ] 返回大厅

## 五、技术需求

| 方面 | 选型 |
|------|------|
| 前端框架 | React + Vite |
| 后端框架 | Node.js + Express |
| 实时通信 | Socket.io |
| 状态管理 | Zustand |
| 包管理 | npm workspaces（monorepo） |
| 类型安全 | TypeScript（前后端共用类型定义） |

### 并发需求
- 最大同时在线约 100 人（约 16~33 局并发游戏）
- 对单一 Node.js 进程压力较小，无需复杂分布式架构

## 七、非功能需求

- **响应速度**：出牌操作到对端展示延迟 < 500ms（局域网/低延迟环境）
- **浏览器兼容**：Chrome / Firefox / Edge 最新版
- **设备**：PC 桌面端（暂不考虑移动端适配）
