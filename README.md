# 🎴 三国杀 3v3 — 网页版统率三军

> 一款基于三国杀官方「统率三军」3v3 竞技模式的网页对战游戏。

三国杀 3v3（统率三军）是三国杀曾经推出的经典竞技模式，现已下架。本项目旨在将这一玩法复刻到网页端，让玩家能够在浏览器中重温这一经典的策略对战体验。

---

## ✨ 功能特性

- 🎮 **完整 3v3 规则**：忠实还原官方统率三军模式的选将、部署、行动单元选择等核心机制
- 🏯 **32 位武将**：覆盖魏、蜀、吴、群四大势力，共 53 个各具特色的武将技能
- ⚔️ **实时对战**：基于 Socket.IO 的低延迟实时对战体验
- 🃏 **108 张卡牌**：包含基本牌、锦囊牌、装备牌，完整的三国杀卡牌池
- 🚪 **房间系统**：输入昵称即可创建或加入房间，无需注册账号
- 🔒 **防作弊设计**：服务端权威架构，所有游戏逻辑在服务端执行，客户端只能看到脱敏后的信息

## 🎯 游戏规则速览

### 基本设定

| 属性 | 说明 |
|------|------|
| 对战人数 | 2 名玩家（每人操控 3 个武将） |
| 阵营 | 暖色方（主公+忠臣×2） vs 冷色方（内奸+反贼×2） |
| 胜利条件 | 击杀对方主帅即获胜 |

### 座次顺序（固定环形）

```
冷色前锋A → 冷色主帅 → 冷色前锋B → 暖色前锋B → 暖色主帅 → 暖色前锋A
```

### 游戏流程

```
创建/加入房间 → 选将（16选8）→ 部署（3个武将分配主帅/前锋）→ 对战 → 胜负结算
```

### 大回合机制

3v3 模式特有的「大回合」行动单元系统：

1. 冷色方选择先手或让先
2. 先手方选择行动单元（主帅行动 / 前锋行动）
3. 按"先→后→先→后"顺序交替执行行动单元
4. 双方各有 1 次主帅行动 + 1 次前锋行动
5. 一个大回合结束后进入下一个大回合

### 武将回合

每个武将的回合依次包含：

1. **准备阶段** — 触发回合开始技能
2. **判定阶段** — 处理延时锦囊（后放先判）
3. **摸牌阶段** — 摸 2 张牌
4. **出牌阶段** — 使用手牌 / 发动技能（每回合限用 1 张杀）
5. **弃牌阶段** — 手牌数降至当前体力值
6. **回合结束** — 触发结束阶段技能

### 3v3 特殊规则

- **击杀奖励**：击杀任意角色立即摸 3 张牌
- **主公技不启用**
- **闪电不使用**
- **南蛮入侵 / 万箭齐发**：出牌方可指定结算方向

### 架构设计

#### 服务端权威

所有游戏逻辑都在服务端执行。客户端只负责：发送用户操作 → 接收脱敏后的游戏状态 → 渲染 UI。

```
客户端操作 → Socket.IO emit → 服务端验证+执行 → 更新 GameState → toClientView() 脱敏 → 广播给所有玩家
```

#### 数据脱敏

服务端通过 `toClientView()` 为每个玩家生成独立的视图：
- 己方武将：可见完整手牌数据
- 对方武将：只能看到 `handCount`（手牌数量），无法看到具体手牌

#### PendingResponse 队列

游戏中所有需要等待玩家输入的操作（出闪、出桃、技能选择、判定介入等）都通过一个异步响应队列 `pendingResponseQueue` 统一管理，支持复杂的链式交互：

```
使用杀 → 等待出闪 → 鬼才介入判定 → 天香转移伤害 → 濒死询问出桃 → ...
```

#### 共享类型

前后端通过 `shared` 包共享类型定义，编译期保证类型安全：

```
shared/src/index.ts → 所有枚举（GamePhase, CardSuit, ResponseType 等）
                    → 所有数据接口（Card, GeneralInstance, GameState 等）
                    → 所有通信协议类型（C2S_*, S2C_*）
```

## 🚀 快速开始

### 环境要求

- **Node.js** >= 18
- **npm** >= 9

### 安装与启动

```bash
# 克隆项目
git clone <repository-url>
cd sgs3v3

# 安装依赖
npm install

# 启动开发服务器（前端 + 后端同时启动）
npm run dev
```

启动成功后：
- 前端：`http://localhost:5174`
- 后端：`http://localhost:3001`

### 游戏体验

1. 在浏览器中打开 **两个标签页**，访问 `http://localhost:5174`
2. 在一个标签页中输入昵称，点击「创建房间」
3. 在另一个标签页中输入昵称和房间码，点击「加入房间」
4. 双方到齐后自动进入选将阶段
5. 选将 → 部署 → 开始对战！

### 构建

```bash
# 生产构建
npm run build
```

## 📡 通信协议

### 客户端 → 服务端（C2S）

| 事件 | 说明 |
|------|------|
| `create_room` | 创建房间 |
| `join_room` | 加入房间 |
| `start_game` | 开始游戏 |
| `pick_general` | 选择武将 |
| `deploy_generals` | 部署武将 |
| `yield_choice` | 选择先手/让先 |
| `choose_action_unit` | 选择行动单元 |
| `use_card` | 使用卡牌 |
| `use_skill` | 使用技能 |
| `respond` | 响应操作 |
| `negate_respond` | 无懈可击响应 |
| `end_turn` | 结束出牌 |
| `discard` | 弃牌 |

### 服务端 → 客户端（S2C）

| 事件 | 说明 |
|------|------|
| `room_created` | 房间已创建（含房间码和玩家 ID） |
| `room_joined` | 已加入房间 |
| `game_state_update` | 游戏状态更新（脱敏后的完整状态） |
| `game_over` | 游戏结束（含胜方信息） |
| `error` | 错误信息 |
